What is EGG

Every word means something.

EGG is not just a name — it's an architecture. Three concepts that together describe a new way to collaborate on open-source software.

E
Event

Every action in EGG — a commit, an issue, a pull request, a comment — is a Nostr event: a signed, verifiable, tamper-proof blob of data distributed across the network.

G
Git

The same Git you already know. EGG doesn't reinvent version control — it replaces the centralized hosting layer. Push, branch, merge, diff: all git operations, all decentralized.

G
Graph

A commit graph — but also a social graph. Contributors, repositories and forks form a web of cryptographically linked events. No central index. No single node of truth. The graph is the network.

How it works

Git operations as Nostr events.

01
kind:30617
Create a repository

A repo is a Nostr event signed with your private key — your npub is your GitHub username. No registration. No email. The repository belongs to your keypair, forever.

02
kind:30618
Push commits as events

Each commit becomes a signed Nostr event carrying the code blob. Published to relays of your choice — or many relays for redundancy. The data is yours, stored where you decide.

03
kind:30619
Open issues and pull requests

Issues and PRs are Nostr events too — signed, threaded, and linked to the repository event. Anyone can comment with their own keypair. No moderator can delete your contribution.

04
relay mesh
Distributed across relays

Events replicate across multiple Nostr relays. If one goes offline, the project survives on others. No single company, government or bad actor can take down your repository.

Event Kinds

The protocol behind the graph.

Every Git concept maps to a Nostr event kind — signed, verifiable, and open.

Event Kind Git Concept Description
kind:30617 Repository Repository metadata: name, description, default branch, topics.
kind:30618 Commit / Patch Code blob signed by the author's keypair. Immutable and verifiable by anyone.
kind:30619 Pull Request Proposed change referencing source and target branch events.
kind:1621 Issue Bug report or feature request linked to a repository event.
kind:1622 Comment / Review Threaded discussion on issues, PRs or specific code lines.
kind:30620 Release / Tag Published release pointing to a signed commit event with changelog.
Why EGG

Open source should be truly open.

No company owns your repo

GitHub, GitLab, Bitbucket — all brilliant, all centralized. They can suspend accounts, remove repos, enforce policies, or simply shut down. EGG has no landlord. Your keypair is your ownership.

Censorship-resistant by design

Free software that criticizes power, security research, whistleblower tools — all vulnerable to platform takedowns. When code is a signed event replicated across thousands of relays, it cannot be silenced.

Identity is your keypair

No email, no verification, no account approval. Your npub is your developer identity — portable across any EGG client, permanent, and self-sovereign. Contributions are cryptographically yours.

Community-operated relays

Anyone can run an EGG relay — individuals, organizations, universities, companies. The more relays, the more resilient the network. No single operator can hold the ecosystem hostage.
