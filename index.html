Mechanism

How EGG works

Three roles. One protocol. No central authority.

01
📡

Node Announces

Compute providers publish their availability, capacity, supported games, and price per tick to the relay network as signed events.

02
🔍

Client Discovers

A lightweight client — phone, browser, or embedded device — subscribes to relays and finds matching nodes for the desired game or application.

03
🤝

Session Negotiation

Client and node exchange signed events to agree on price, tick rate, and session parameters. The node issues a session ID and WebSocket endpoint.

04
🎮

Live Execution

The node runs the Execution Packet (WASM game logic) tick by tick. Delta-encoded state updates flow to the client. Inputs flow back in real time.

05
⚡

Lightning Payment

After N ticks, the node issues a BOLT-11 invoice. The client pays millisatoshis via the Lightning Network. The preimage is returned as proof.

06
🔄

Offline Resilience

If connectivity drops, the client caches state and queues inputs. Sessions resume automatically when the connection is restored, within a grace window.

Technical Design

Protocol Architecture

EGG is relay-mediated but not relay-dependent. Nodes and clients can communicate directly; relays provide discovery and coordination.

Economic Layer

Native Lightning Economy

EGG's economic layer uses Bitcoin's Lightning Network as a first-class primitive — not an afterthought. Payments are peer-to-peer, instant, and measured in millisatoshis.

⚡

Pay-Per-Tick

Billing is tied to discrete execution ticks (~100ms). You pay only for compute you actually consume. Sessions can pause without accumulating debt.

msat / tick
🌐

Permissionless

No bank account required. Anyone with a Lightning wallet can participate — as a player paying sats, a node earning sats, or a creator receiving royalties.

global access
🎨

Creator Royalties

Execution Packets embed a creator Lightning address and royalty percentage. Nodes route a share of collected sats to creators on every session played.

royalty_pct
🆓

Free Tier

Nodes may set price_msat_per_tick to zero. Educational content, public interest applications, and community nodes can operate completely free of charge.

0 msat/tick

Event Model

Protocol Events

Every action in EGG is a cryptographically signed event. Below are the core event schemas.

{ "id": "<SHA-256 of canonical fields>", // event fingerprint "pubkey": "<secp256k1 public key, hex>", // sender identity "created_at": 1745000000, // Unix timestamp "kind": 10000, // event type "tags": [["name", "value"], ...], "content": "", "sig": "<Schnorr sig over id, BIP-340>" // authentication }
{ "kind": 10000, // Node Announcement "tags": [ ["name", "node-amazonia-01"], ["capacity", "8"], // max concurrent sessions ["games", "snake-v1", "pong-v2", "quiz-geo-br"], ["latency_region", "sa-east"], ["price_msat_per_tick", "10"], // 0 = free ["endpoint", "wss://node.example.com/egg"], ["expires", "1745000000"] ] }
{ "kind": 10100, // Session Request "tags": [ ["game", "snake-v1"], ["node", "<node_pubkey>"], ["client_version", "egg-client/0.1"], ["payment_capability", "lightning"], ["max_budget_msat", "5000"] // session budget cap ] }
{ "kind": 10201, // State Update (Node → Client) "content": "<base64-encoded delta JSON>", "tags": [ ["session_id", "<uuid-v4>"], ["tick", "4821"], // sequential tick counter ["encoding", "delta_json"], // or "full_json" ["checksum", "<sha256_full>"] // for resync verification ] }
{ "kind": 10300, // Payment Request "content": "lnbc10n1...", // BOLT-11 Lightning invoice "tags": [ ["session_id", "<uuid-v4>"], ["ticks", "100"], // ticks covered ["amount_msat", "1000"], // 10 msat × 100 ticks ["expires", "1745001000"] // invoice expiry ] }

Build on EGG

Developer Adoption

EGG is an open specification. No permission required. Build nodes, relays, clients, or execution packets.

To build a conformant EGG implementation, you need:

  • WebSocket client/server library
  • secp256k1 + Schnorr (BIP-340) signing
  • SHA-256 event ID computation
  • JSON serialization / deserialization
  • Lightning Network wallet or library (BOLT-11)
  • WebAssembly runtime (for node execution)

Compatible languages and runtimes:

  • JavaScript / Node.js — WebSocket, secp256k1, WASM
  • Python — asyncio, coincurve, wasmtime
  • Go — gorilla/websocket, btcec, wazero
  • Rust — tokio-tungstenite, secp256k1, wasmer
  • C / C++ — libwebsockets, libsecp256k1
  • Flutter / Dart — mobile clients, embedded Lightning
